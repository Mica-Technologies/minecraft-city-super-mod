# Dynamic Highway Guide Sign — Implementation Plan & Progress

Player-configurable MUTCD highway guide signs (green/blue/brown freeway signs with route shields,
exit tabs, arrows, destination text). Inspired by
[SignMaker](https://github.com/JKPotato-Computer/SignMaker). Rendered via TESR, configured through
a multi-tab scrollable GUI. Block registry name: `dynamic_guide_sign`, GUI ID: 14.

---

## Architecture Summary

- **Data model:** `trafficaccessories/guidesign/` package. `GuideSignData` root serialized as JSON
  in a single NBT key (`"signData"`). Hierarchy: GuideSignData > GuideSignPanel > GuideSignRow >
  GuideSignElement (polymorphic: TEXT, SHIELD, ARROW, DIVIDER, SPACING).
- **Block:** `BlockDynamicGuideSign` extends `AbstractBlockRotatableNSEW` + `ICsmTileEntityProvider`.
  Material IRON, SoundType METAL, CUTOUT_MIPPED render layer.
- **TileEntity:** `TileEntityDynamicGuideSign` extends `AbstractTileEntity`. Lazy JSON deserialization
  with `stateDirty` flag. Expanded render bounding box (pos +/-8, +/-4).
- **Network:** `DynamicGuideSignUpdatePacket` + `DynamicGuideSignUpdateHandler`. Max 8KB JSON.
  Registered for `Side.SERVER`.
- **Renderer:** `TileEntityDynamicGuideSignRenderer` — TESR with direct rendering (no display list
  cache currently). Uses raw GL11 calls alongside GlStateManager to avoid cache desync.
- **GUI:** `DynamicGuideSignGui` — 3-tab GuiScreen (Properties, Panel Editor, Row Element Editor)
  with scroll support via `Mouse.getEventDWheel()`.
- **Atlas:** Single combined 512x512 texture (`sign_atlas.png`) with 64x64 cells. Shields in row 0
  (cols 0-7), arrows in rows 4-5 (cols 0-4). Generated by `GuideSignAtlasTool` in dev-env-utils
  using Apache Batik for SVG rendering.

## Phase 3: TESR Renderer — IN PROGRESS

### Completed
- [x] Basic render pipeline: pushMatrix, translate, rotate by facing, scale(0.0625)
- [x] Sign background rendering (border + colored face as layered cuboids)
- [x] Exit tab rendering (separate Z layer to avoid z-fighting with sign face)
- [x] Text element rendering (FontRenderer at fullbright 240,240)
- [x] Shield element rendering (atlas texture quad + route number overlay + banner text)
- [x] Arrow element rendering (atlas texture quad)
- [x] Post rendering (5 types: LEFT, RIGHT, CENTER, OVERHEAD=none, RURAL=two)
- [x] Panel divider rendering
- [x] Dynamic dimension computation from content
- [x] GL state management: raw GL11 calls alongside GlStateManager to prevent cache desync
- [x] Z-ordering fix: sign face at faceZ-0.1 renders in front of border
- [x] Exit tab anchored to signTop (not panelY) so it sits above the sign
- [x] Banner text positioned within row bounds (0.5 offset, 0.35 scale)
- [x] Proportions overhaul (thin border 0.4, tight padding, realistic dimensions)

### Remaining
- [ ] **Corner style rendering** — ROUND vs SHARP corners (currently ignored, always sharp)
- [ ] **Sign width minimum** — signs with very short text may look too narrow; may need a
      configurable minimum width or auto-sizing that accounts for shield+text combinations
- [ ] **Multi-panel layout** — verify panel dividers and multi-panel stacking looks correct
      with the new proportions
- [ ] **Text alignment** — currently all rows are centered; real signs may need left-aligned
      text for destination names
- [ ] **Further proportions tuning** — may need adjustment after more in-game testing with
      various sign configurations (different text lengths, multiple shields, etc.)
- [ ] **Display list caching** — currently removed due to GlStateManager desync issues.
      Phase 5 item: reintroduce using raw GL11 calls (not GlStateManager) for all state
      changes within the display list compilation. The `stateDirty` flag and
      `cleanupDisplayList()` infrastructure remains in TileEntity.

## Key Bugs Fixed

| Bug | Root Cause | Fix |
|-----|-----------|-----|
| Sign face renders white | Border cuboid's front face (Z+0.1) occluded sign face cuboid (Z+0) | Sign face rendered at Z-0.1, in front of border |
| Colors on back of sign | Z offset was +0.1 (toward back), should be -0.1 (toward front) | Flipped Z offset direction |
| Overhead mode shows two posts | OVERHEAD case drew two vertical posts | Changed to empty break (no posts) |
| Exit tab clips into sign | Tab anchored to panelY (inside content area) | Changed to signTop (sign edge) |
| Banner text overflows sign | Banner Y offset 3.0 above shield exceeded row bounds | Reduced to 0.5 offset, 0.35 scale |
| Exit tab z-fights with sign | Tab and sign face at same Z depth | Tab uses separate Z layer (faceZ-0.2) |
